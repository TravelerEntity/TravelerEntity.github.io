线程和进程

当我们在一边写代码一边听音乐的时候，可以大概描述成现在正在执行两个进程——听音乐和写代码，然后写代码的这个进程下又有多个线程，比如脑中思考代码的线程，手里敲键盘的线程，看屏幕上已有代码的线程等。但其实人脑并不具备多进程工作的能力，我们只是快速地在多个任务之间来回切换，多进程工作的能力只是一种错觉。

程序默认的执行逻辑是从上到下，如果在中间调用了其它的方法则主线程去执行其它该方法，执行完后返回到主方法继续顺序执行。如果开启多线程的话，程序在碰到其它方法的调用时，会新增一个线程去独立执行，原来的主线程还是自己做自己的。

关于多线程的核心概念

  1. 程序运行时会默认的创建一些线程，比如 Main() 线程。
  2. 多个线程之间的执行顺序由CPU决定
  3. 多线程会带来额外的开销，反复横跳肯定要付出额外的资源的呀
  4. 一个进程就是一个程序的单次运行结果，而一个进程下也允许有多个线程共同协作。

继承Thread类开启线程

  1. 自定义一个继承自 Thread 的类
  2. 重写它的 run()方法
  3. 主函数中调用该类实例的 start() 方法即可开启线程。很简单是吧？（y1s1,qs…

另外，开启的线程并不一定会立即执行，这个是看cpu的调度，我们无法控制，核心概念第二条提到过。

public class MainTest 
{
    public static void main(String[] args) 
    {
        DemoExThread d = new DemoExThread();
        d.start();
        for (int i = 0; i < 400; i++) 
        {
            System.out.println("This is Main Thread_____________"+i);
        }
    }
}

class DemoExThread extends Thread
{
    @Override
    public void run() 
    {
        for (int i = 0; i < 100; i++) 
        {
            System.out.println("这肯定不是主线程~"+i);
        }
    }
}

实现Runnable接口开启线程

实现 Runnable 接口来开启线程的方式相比继承 Thread 类只多了第四步，两种方式的接口是一样的。只是在实际使用中，更推荐实现 Runnable 接口来开启线程。因为 Java 不是只支持单继承嘛，继承的名额要物尽其用。

  1. 自定义一个实现了 Runnable 接口
  2. 重写 run() 方法
  3. 然后在主函数中实例化该类对象
  4. 实例化一个 Thread 对象，并把 Runnable 接口的实现类传进它的构造参数
  5. 调用 Thread 对象的 start() 方法开启线程（因为Runnable 接口没有 start() 方法，曲线救国，曲线救国…）。
public class MainTest 
{
    public static void main(String[] args) 
    {
        DemoImRunnable dr = new DemoImRunnable();
        new Thread(dr).start();
        for (int i = 0; i < 200; i++) 
        {
            System.out.println("This is the Main thread*"+i);
        }
    }
}
class DemoImRunnable implements Runnable
{
    @Override
    public void run()
    {
        for (int i = 0; i < 100; i++) 
        {
            System.out.println("这肯定不是主线程*"+i);
        }
    }
}
